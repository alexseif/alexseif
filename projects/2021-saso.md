# Use Case: Mitigating Concurrency Collisions in a Distributed Booking Engine

### 1. Objective & Scope
Led the 3-year migration and optimization of Saudi Arabia's nationwide vehicle inspection and modification scheduling system. The system manages immense daily volume, orchestrating vehicle service lanes across the country into tight 15-minute scheduling blocks.

### 2. Technical Decisions & Implementation
* **The 3-Client Concurrency Crisis:** Solved a critical race-condition bottleneck where three separate clients—the public web portal, the internal government employee backend, and the citizen mobile app—competed simultaneously for the exact same 15-minute time slots.
* **Distributed Time-Slot Lock System:** Designed an API lock-state system that pushed time-slot selection to the absolute *last* step of the checkout workflow. The engine granted a strict 5-minute buffer hold on a selected slot; if the user failed to complete the form fields within 5 minutes, the state machine released the slot instantly back to the public pool to maximize capacity.
* **Database Restructuring (3NF + Star Schema):** Eliminated heavy database latency under peak loads by refactoring fragmented tables into 3rd Normal Form (3NF) for transactional write safety. Paired this with a Star Schema design for rapid read queries, allowing dense reservation calendars and facility drop-downs to render instantly.
* **Headless React Migration:** Led the multi-phase evolution of the legacy monolithic Drupal core into a decoupled, headless ecosystem. Exposed the core business logic via versioned APIs to a high-performance React frontend, isolating user traffic from the backend workflows.
* **Operational Shift for Proxies:** Modified checkout form state logic to allow Indian and foreign-national drivers to log legally validated proxy authorizations on behalf of car owners without breaking tracking security.
* **Technical Team Governance:** Standardized engineering communication across the project by writing a project-wide terminology dictionary. Mapped strict Jira workflows to engineering release calendars to stop unplanned hotfixes and prevent weekend system breaks.
