# Use Case: Dynamic Geo-Location Routing for National Fleet Inspections

### 1. Objective & Scope
The Egyptian Ministry of Interior required a nationwide web ecosystem to offload vehicle inspections from crowded, traditional traffic police departments to new mobile inspection vans, commercial hubs, and shopping malls. 

### 2. Technical Decisions & Implementation
* **Multi-Phase Architecture Evolution:** Directed the portal’s growth from a basic informational web presence into a highly interactive, operational booking platform across a multi-year development timeline.
* **Back-to-Back API Integration:** Engineered a secure, high-availability API connection linking the Drupal frontend directly with the state-level backend database, enabling citizens to find and track mobile inspection units in real time.
* **Dynamic Location Matching:** Developed relational scheduling filters that allowed users to input their required vehicle service and immediately find the closest mobile van or commercial kiosk possessing the exact tools and capacity to perform that specific inspection.
* **Citizen Feedback Loop:** Built an integrated support and data-collection system within the portal to instantly capture, route, and log citizen issues arising during field operations, protecting service reliability.
